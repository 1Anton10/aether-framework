Guide

Aether компилирует UI в Wasm: состояние в памяти, обновления — dirty DAG и binary DSM, async — через suspend/resume effects. Ниже — всё для работы с v1.0.

Обзор

JSX / Vue / Svelte / Angular / Solid / Qwik / Lit / HTML
        │
        ▼
 aether-compile → IR → app.wasm (+ app.gc.wasm)
        │
        ▼
 snapshot → mount / SSR hydrate → DOM patches → DSM

Рантайм не гоняет Virtual DOM. Слоты лежат в linear (и опционально WasmGC) памяти; рёбра IR связывают слот с text/attr; сервер и клиент обмениваются бинарными дельтами.

Установка

Из git (полный контроль, всегда работает):

git clone https://github.com/1Anton10/aether-framework.git
cd aether-framework
npm install
npm run setup
npm run create -- my-app
cd my-app && npm run dev

С npm (после релиза v1.0.0): npm create aether@latest my-app

Сервер: порт из aether.config.json (demo-сайт фреймворка — 3000).

UI и entry

Entry задаётся в aether.config.json → entry. В JSX допустимы статические атрибуты, onClick={handler}, текст {slotName}. Имена {slot} и handlers должны совпадать с bindings / state. 

// src/App.tsx
export default function App() {
  return (
    <div>
      <h1>{count}</h1>
      <button onClick={inc_count}>+</button>
    </div>
  );
}
// Vue SFC — src/App.vue
<template>
  <div>
    <h1>{{ count }}</h1>
    <button @click="inc_count">+</button>
  </div>
</template>

State и bindings

src/state.ts — начальные значения слотов (числа).

export const initialState = {
  count: 0,
  doubled: 0,
};

aether.bindings.json — derived (пересчёт по DAG) и handlers. Схема: schemas/bindings.schema.json. 

{
  "derived": [
    { "target": "doubled", "from": "count", "op": "mul", "arg": 2 }
  ],
  "handlers": {
    "inc_count": { "op": "inc", "slot": "count" }
  }
}

Derived ops: mul, add, copy. Изменение источника помечает dirty-цепочку.

Handlers

opПоведение
inc / dec / addЛокальный mutate слота (delta)
setАбсолютное значение (delta = value)
server: truePOST /api/delta — сервер применяет дельту
performEffect: suspend → host → resume в into
 
{
  "handlers": {
    "add_item": { "op": "inc", "slot": "cart", "delta": 1 },
    "sync_cart": { "op": "inc", "slot": "cart", "server": true, "delta": 1 },
    "load_catalog": { "op": "perform", "effect": "db.get", "into": "catalog" },
    "measure_ping": { "op": "perform", "effect": "ping", "into": "pingMs" }
  }
}

Effects

В конфиге effects — серверные обработчики имён effect. Клиент всегда шлёт POST /api/effect; для rtt в слот пишется wall-clock RTT. 

// aether.config.json
"effects": {
  "db.get": { "type": "catalog" },
  "ping": { "type": "rtt" }
}
typeРезультат в слот
set / fixedarg
add / mulот текущего payload
catalogдлина серверного каталога; список — GET /api/catalog
rttмс round-trip клиента

Pages и loaders

src/pages/index.tsx        → /
src/pages/about.tsx        → /about
src/pages/about.loader.ts  → server loader
→ dist/pages.manifest.json + #aether-loaders

Loader — модуль на сервере; данные отдаются в HTML как JSON и читаются на клиенте. Это не полный Nuxt data layer: адаптируйте fetch под свой бэкенд.

SSR

При "ssr": true сервер рендерит IR в HTML с data-aether-nid и встраивает snapshot. Клиент гидратирует по nid, не очищая #root. Stream: GET /api/ssr.

DSM (binary sync)

  • POST /api/delta — кадр [slot u32][len u32][payload…]
  • WS /aether-dsm и /api/dsm — тот же кодек (приём/broadcast)
  • Mutate с клиента в demo идёт по HTTP, чтобы вернуть размер ответа

Контракт: docs/ABI.md.

Синтаксисы и совместимость

Aether — не runtime Vue/React. Компилятор парсит синтаксис в общий IR (подмножество шаблона). Meta-фреймворки (Nuxt, Next, SvelteKit) поддерживаются через migrate структуры проекта + тот же парсер синтаксиса базы (Vue / JSX / Svelte), а не через запуск их серверов как есть.

ВходПарсерMigrateЗамечание
React / Preact JSX SWC JSX Next, Vite React Базовый путь; полный React runtime не эмулируется
Vue SFC .vue <template> → IR Vue, Nuxt {{x}}, @click, :attr, v-for → Loop
Svelte .svelte markup → IR Svelte, SvelteKit {x}, on:click, {#each} → Loop
Angular template .component.html / маркеры Angular {{x}}, (click), *ngFor → Loop
Solid JSX детект solid-js Solid {count()} → слот; items().map → Loop
Qwik JSX детект component$ Qwik обёртки упрощаются к дереву
Lit html`…` Lit template literal → IR
HTML шаблон — интерполяции + click-атрибуты

Форки и meta: Next / Remix / Gatsby → JSX + migrate pages; Nuxt / Weex / Pinia → Vue-парсер; SvelteKit / Elder → Svelte; Analog → Angular; Astro / Alpine → detect + HTML/JSX entry. Отдельного «чужой runtime inside Aether» нет — после migrate вы на Aether CLI. Pinia/Redux не эмулируются: стейт = Wasm-слоты + derived + effects.

Пакеты-мосты: aether-compat-react|vue|svelte|solid|qwik|lit|angular.

Migrate

# Next / Remix / Gatsby / Nuxt / Vue / SvelteKit / Analog / Elder / Astro / Alpine / …
npm run migrate

Пишет aether.config.json, копирует pages в src/pages при наличии, переписывает импорты на compat. Data-fetching нужно вручную перевести на loaders / effects.
